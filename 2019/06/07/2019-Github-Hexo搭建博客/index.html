    • 2019 Github+Hexo搭建博客

    最近起了搭建一个博客的想法，一来想在网络上有我们的一个小地方，可以放一些我们的输出，二来也许借这个地方认识一些朋友也说不定（期待）。查了些资料，还是比较中意Github平台的搭建方式，本来就有Github的账号，对博客呈现的完全控制更是非常吸引我。上手折腾了几天，可算完成搭建，因为实属这方面的小白，中间走了有些弯路，觉得有必要记录下来，供有同样想法的朋友参考，如下：

    一、前期准备：

    参考我的情况，需要有以下前期准备：

    1. 需要有关于Git的一般知识 & Github账号；
    2. 需要联网操作；
    3. 需要安装Git / TortoiseGit；

    二、环境安装：

    总体思路是：

    1. 使用Github的博客服务Github Pages创建一个特殊的Repository，它的作用在于将此Repository中的信息生成一个网页并展示出来，也就是我们的博客页面，理解成是一个博客展示器？（但内容我们说了算）；
    2. 安装Hexo博客框架 / 工具，可以在本地生成博客，当然，生成的博客内容我们需要发布到前面Github Pages的那个Repository里，让大家可以看到；
    3. 用Hexo创建本地博客；
    4. 把本地博客部署到Pages；

    That’s it，因为工具给力，所以还蛮简单，开整！

    2.1 创建Github Pages仓库(Repository)

    用自己的账号登陆Github，在Repository界面中创建一个以github.io结尾的Repository即可，如solidstateyu.github.io，Github会识别到这个特殊的标记，将这个Repository视为Pages来处理，如下图：

    记得勾选“Initialize this repository with a README”。

    创建完成后，打开网页：http://solidstateyu.github.io 就可以看到ReadMe里面的内容了。

    所以说Github Pages就是个博客展示器，因为新创建的Repository空荡荡的只有个ReadMe，所以Pages也就只能呈现这么多内容了。

    下面就请出Hexo，帮我们构建可以呈现更丰富内容的博客。

    2.3 创建本地博客

    在电脑上找个路径，创建一个文件夹，如D:\MyBlog。右键该文件夹，选择Git Bash Here，呼出Git终端。

    输入hexo init将这个文件夹初始化成一个博客文件夹；

    输入npm install安装依赖包：

    输入hexo g生成本地博客（Hexo自己带了一个Hello World博客）

    输入hexo s将本地博客放到本地服务器上，供我们浏览，打开浏览器，输入http://localhost:4000/ 就可以看到了。

    回到Git终端，按下Ctrl+C就可以停止这个服务器，结束本地浏览了。

    至此，我们有了一个本地的博客。先什么都不改，把它部署到网络上去吧~

    四、发布博文

    OK，博客搭建好了，是时候写下第一篇博文。

    发布博文的一般流程是：

    1. 用Hexo新建一个空白博文；
    2. 用Markdown编辑器编辑这个博文；
    3. 用Hexo生成且部署博文到Github的Repository；

    三步走完，博文就可以在博客里看到了。分解动作来一遍：

    打开Git终端，输入hexo new “第一篇博文”，就可以在D:/MyBlog/source/posts/目录下看到一个md格式的空白博文文件了。

    用各种编辑器（比如我正在用的typora），编辑这份文件，完事记得保存；

    回到Git终端，输入hexo d -g，完成生成和部署。

    当然，也可以先本地预览一下，看看效果，然后再部署，那就：

    1. hexo g，生成博文
    2. hexo s，本地浏览一下(在本地浏览器中输入localhost:4000即可看到)
    3. hexo d，部署到Git Repository
